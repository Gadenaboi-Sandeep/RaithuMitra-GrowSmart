1. APPLICATION OVERVIEW
--------------------------------------------------------------------------------

Name          : RaithuMitra (రైతు మిత్ర)
Meaning       : "Farmer's Friend" in Telugu
Type          : Native Android Mobile Application
Target Region : Andhra Pradesh / Telangana, India (Telugu-language localized)
Purpose       : A digital marketplace and management platform designed to
                bridge the gap between farmers, agricultural equipment owners,
                agricultural laborers/workers, and corporate agri-businesses.
                The app enables contract farming, equipment rental, and labor
                hiring — all through a unified mobile platform.

Tagline       : Empowering Farmers, Connecting the Agricultural Ecosystem

--------------------------------------------------------------------------------
2. PROBLEM STATEMENT
--------------------------------------------------------------------------------

Indian farmers — especially small and medium land-holders — face persistent
challenges:
  - Difficulty finding and renting agricultural equipment at affordable rates.
  - Lack of a structured system to hire reliable agricultural laborers.
  - No direct channel between farmers and corporates for contract farming.
  - Manual, unorganized processes that lead to inefficiency and financial loss.

RaithuMitra addresses all these challenges through a single, role-based mobile
application available in the Telugu language for accessibility.

4. CORE FEATURES & MODULES
--------------------------------------------------------------------------------

4.1 AUTHENTICATION MODULE
--------------------------
  - User Registration (Signup) with role selection:
      FARMER, CORPORATE, WORKER, OWNER
  - User Login with JWT-based authentication
  - Persistent session management via Android SharedPreferences
  - Secure token storage and automatic injection into API calls
  - Auto-logout on session expiry / manual logout from Navigation Drawer
  - User profile data (name, phone, address) persisted locally

4.2 HOME / DASHBOARD (Landing Page)
-------------------------------------
  - Role-specific dashboard shown after login
  - Summary cards showing:
      • NUMBER OF ACTIVE CONTRACTS
      • AVAILABLE EQUIPMENT count
      • AVAILABLE LABORERS count
      • PENDING REQUESTS count (for Owners)
  - Quick-action navigation to all modules
  - Swipe-to-Refresh support for live data reload
  - Equipment booking request management for Owners (Accept/Reject)
  - Labor hiring request management for Workers

4.3 CONTRACT MODULE (ఒప్పందాలు)
----------------------------------
  - Browse available contract farming opportunities
  - Create new contract proposals (Corporate users)
  - View contract details: crop type, location, duration, payment terms
  - Sign contracts digitally (one-click signing)
  - Local caching of contracts via Room database
  - Sync with remote server on every load

4.4 EQUIPMENT MODULE (అద్దె / ఇన్వెంటరీ)
-------------------------------------------
  - Browse available agricultural equipment for rent
  - Equipment details: type, owner name, price, availability
  - Book equipment by submitting a booking request
  - Add new equipment listings (Owner role)
  - Upload equipment images via multipart file upload
  - Delete own equipment listings
  - Owner dashboard for managing received booking requests:
      Status flow: PENDING → CONFIRMED / REJECTED
  - Local caching via Room database

4.5 LABOR MODULE (కూలీలు / పనులు)
-------------------------------------
  - Browse available agricultural laborers
  - View laborer profiles: name, skill set, daily rate, availability
  - Hire laborers by submitting hiring requests
  - Create / register a labor profile (Worker role)
  - Owner/Farmer view of received hiring requests with Accept/Reject
  - Status tracking: PENDING → CONFIRMED / REJECTED
  - Local caching via Room database

4.6 PROFILE MODULE
-------------------
  - View personal profile: name, phone number, address
  - Accessible via the Navigation Drawer
  - Data is fetched from session and displayed

4.7 NAVIGATION
---------------
  - Bottom Navigation Bar — role-aware, shows only relevant tabs:
      FARMER  : Home | Contracts | Equipment | Labor
      CORPORATE: Home | Contracts
      WORKER  : Home | Labor (Jobs)
      OWNER   : Home | Equipment (Inventory)
  - Side Navigation Drawer with:
      Profile link
      Logout button
      User name & phone number in header

10. KEY DESIGN DECISIONS
--------------------------------------------------------------------------------

  1. OFFLINE-FIRST CACHING:
     Room database is used to cache contracts, equipment, and laborers locally.
     This allows users in areas with intermittent connectivity to still view data.

  2. ROLE-BASED UI:
     The bottom navigation bar is built programmatically based on the user's
     role stored in SharedPreferences, ensuring a clean, uncluttered experience.

  3. MVVM ARCHITECTURE:
     Separates UI logic from business logic. ViewModels survive configuration
     changes (e.g., screen rotation) and provide LiveData streams to the UI.

  4. REPOSITORY PATTERN:
     Repositories abstract the data source (remote vs. local), making it easy
     to add new sources or mock data for testing.

  5. JWT AUTHENTICATION:
     Stateless authentication ensures the server doesn't maintain session state,
     making the backend horizontally scalable.

  6. TELUGU LOCALIZATION:
     Full Telugu-language UI ensures accessibility for rural farmers who may
     not be comfortable with English-only interfaces.

11. DEVELOPMENT SETUP
--------------------------------------------------------------------------------

  Android App:
    1. Open project in Android Studio
    2. Sync Gradle dependencies
    3. Configure backend URL in RetrofitClient.java (Base URL)
    4. Build and run on an Android emulator or physical device

  Backend Server:
    1. Ensure MySQL is running and the database is created
    2. Configure application.properties with DB credentials
    3. Navigate to /backend directory
    4. Run: mvn spring-boot:run
    5. Server starts on http://localhost:8080

  Network:
    - For physical device testing, ensure the device and PC are on the same
      network and update the base URL to the PC's local IP address.
